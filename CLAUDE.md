## Commands

```bash
# Start the server
npm start

# Run all tests
npm test

# Run tests in watch mode
npm run test:watch

# Run a single test file
npx jest tests/controllers/bars.test.js

# Run tests with coverage
npm run test:coverage

# Fresh database setup (drops and recreates schema)
npm run setup

# Initialize the database schema only
npm run setup-db
```

## Environment Variables

Copy `.env.example` to `.env` before running locally. Required variables (validated at startup in `config/env.js`):

- `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_PORT` ‚Äî MySQL connection
- `PORT` ‚Äî server port
- `JWT_SECRET` ‚Äî secret for signing JWTs
- `REACT_APP_API_URL` ‚Äî allowed CORS origin for the frontend

## Architecture

**Entry point:** `server.js` ‚Üí `src/app.js` ‚Üí routes ‚Üí controllers

**Layer structure (no ORM ‚Äî raw SQL via mysql2/promise):**
- `src/app.js` ‚Äî Express setup, CORS, route mounting
- `src/routes/` ‚Äî Express Router files, define public vs. protected endpoints
- `src/controllers/` ‚Äî All business logic and SQL queries
- `src/utils/db.js` ‚Äî Singleton mysql2 connection pool (use `db.execute()` for prepared statements with user input, `db.query()` for dynamic SQL)
- `src/utils/auth.js` ‚Äî JWT middleware: `authenticateToken` (required) and `optionalAuth` (permissive)
- `src/utils/eventRecurrence.js` ‚Äî Logic for generating recurring event instances
- `config/database.js` ‚Äî DB config object (reads from env)
- `config/env.js` ‚Äî Loads `.env` and warns on missing required vars

**Route modules and their controllers:**

| Mount | Route file | Controller |
|-------|-----------|------------|
| `/bars` | `src/routes/bars.js` | `src/controllers/bars.js` |
| `/events` | `src/routes/events.js` | `src/controllers/events.js` |
| `/tags` | `src/routes/tags.js` | `src/controllers/tags.js` |
| `/event-tags` | `src/routes/event-tags.js` | `src/controllers/event-tags.js` |
| `/users` | `src/routes/users.js` | `src/controllers/users.js` |
| `/app-users` | `src/routes/app-users.js` | `src/controllers/appUsers.js` |

**Two distinct user types:**
- `web_users` ‚Äî admin/internal users (role: `super_admin`), managed via `/users`
- `app_users` ‚Äî customer-facing accounts, managed via `/app-users`; JWT tokens include `userType: 'app_user'`; the `appUsers.js` controller checks this claim with `ensureAppUserToken()`

**Database conventions:**
- All primary keys are UUIDs (via `uuid` package)
- Soft deletes use `is_active = 0` (never hard-delete bars, events, or app users)
- `bar_hours.crosses_midnight` is auto-calculated on write ‚Äî don't trust the client value
- Events use a master record + `event_instances` rows (generated by `eventRecurrence.js`)
- Tags for bars: `bar_tags` table + `bar_tag_assignments` join table
- Tags for events: `event_tags` table (separate from bar tags)

**Query patterns:**
- Related data (hours, tags, events) is included via `?include=hours,tags,events` query param
- Tags and events are aggregated with `GROUP_CONCAT` in a single query, then parsed in JS
- Geolocation filtering and distance sorting uses the Haversine formula inline in SQL
- Pagination uses `page`/`limit` or raw `offset`; `limit` must be 10‚Äì100

**Tests:**
- Tests live in `tests/` with subdirs `controllers/`, `Routes/`, `Utils/`, and `helpers/`
- `tests/setup.js` is the Jest `setupFilesAfterEnv` file ‚Äî sets env vars and global timeout
- `tests/helpers/authHelpers.js` contains shared auth utilities for tests
